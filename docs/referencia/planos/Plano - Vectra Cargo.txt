# (Referência) Plano Original: Sistema Multi-Agente

## Contexto Original

O Cargo Flow Navigator possui workflows manuais (cotações, ordens de serviço, financeiro) sem automação entre etapas. Todas as transições de stage dependem de ação humana no frontend, o que gera:
- Atrasos na comunicação com clientes e motoristas
- Documentos financeiros (FAT/PAG) criados manualmente
- Nenhuma análise inteligente de rentabilidade
- Sem fluxo de aprovação gerencial
- Notificações apenas via toast no browser (nada por email/WhatsApp)

**Objetivo**: Criar um sistema multi-agente (Hierarchical + Human-in-the-loop) que automatize todo o pipeline logístico.

**Stack de integração**: Supabase (Event Bus + Edge Functions) + **OpenClaw** (AI Agent + WhatsApp + Orchestration Skills) + Claude API (Análise Financeira)

---

### 1.3 — AI Financial Analysis Agent

**Nova Edge Function**: `supabase/functions/ai-financial-agent/index.ts`

Usa a Anthropic REST API diretamente (Deno runtime):
```
POST /ai-financial-agent
Body: {
  analysisType: 'quote_profitability' | 'financial_anomaly' | 'approval_summary' | 'dashboard_insights'
  entityId: string
  entityType: string
}
```

**Tipos de análise:**

1. **quote_profitability** — Quando cotação chega em `precificacao`:
   - Compara valor vs piso ANTT
   - Analisa margem vs média histórica
   - Retorna: risco (baixo/médio/alto), margem %, recomendação

2. **financial_anomaly** — Quando documento financeiro é criado:
   - Compara montante vs faixa histórica para rotas similares
   - Verifica condições de pagamento incomuns
   - Retorna: flags de anomalia, severidade

3. **approval_summary** — Quando `approval_request` é criado:
   - Resume dados da entidade para o aprovador
   - Destaca fatores de risco
   - Fornece recomendação aprovar/rejeitar com justificativa
   - Salvo em `approval_requests.ai_analysis`

4. **dashboard_insights** — Diário via pg_cron:
   - Analisa últimos 30 dias
   - Identifica tendências, taxa de conversão, gargalos
   - Salvo em tabela `ai_insights`

**Nova tabela:**
```
ai_insights
├── id (UUID PK)
├── insight_type (TEXT)
├── entity_type / entity_id (TEXT/UUID)
├── analysis (JSONB) — { risk, margin, recommendation }
├── summary_text (TEXT)
├── expires_at / created_at (TIMESTAMPTZ)
```

### 1.4 — Workflow Orchestrator

**Nova Edge Function**: `supabase/functions/workflow-orchestrator/index.ts`

Chamada via **Database Webhook** (Supabase) no INSERT da tabela `workflow_events`.

**Lógica central:**
1. Busca eventos `status = 'pending'` (batch de 20)
2. Marca como `processing`
3. Roteia por `event_type` para o handler correto
4. Sucesso → `completed` | Falha → retry ou `failed`

**Tabela de roteamento:**
| event_type | Condição | Ações |
|------------|----------|-------|
| `quote.stage_changed` → `ganho` | sempre | 1) Auto-criar OS (RPC) 2) Auto-criar FAT 3) Notificar cliente |
| `quote.stage_changed` → `precificacao` | sempre | 1) Análise AI de rentabilidade |
| `order.created` | sempre | 1) Auto-criar PAG (se tem carreteiro) |
| `order.stage_changed` → `busca_motorista` | driver_phone != null | 1) WhatsApp para motorista (via OpenClaw) |
| `order.stage_changed` → `entregue` | sempre | 1) FAT → GERADO 2) PAG → GERADO 3) Notificar cliente |
| `document.uploaded` | tipo POD/NFe/CTe | 1) Checar requisitos → auto-avançar stage |
| `financial.status_changed` → `GERADO` | total > threshold | 1) Criar approval_request 2) AI analysis 3) Notificar aprovador |
| `financial.status_changed` → `GERADO` | total ≤ threshold | 1) Auto-avançar para AGUARDANDO |
| `approval.decided` = approved | sempre | 1) Auto-avançar documento financeiro |
| `installment.overdue` | pg_cron diário | 1) Email + WhatsApp de cobrança |

### 2.2 — OpenClaw Integration (WhatsApp + AI Jobs)

**OpenClaw Skills a criar:**

1. **`cargo-whatsapp-notifier`** — Skill que recebe webhooks do Supabase e envia WhatsApp
```
   openclaw cron add \
     --name "Check pending notifications" \
     --cron "*/1 * * * *" \
     --message "Buscar notification_logs pendentes em Supabase e enviar via WhatsApp"
```

2. **`cargo-overdue-checker`** — Job diário de cobranças
```
   openclaw cron add \
     --name "Daily overdue check" \
     --cron "0 9 * * *" \
     --tz "America/Sao_Paulo" \
     --message "Verificar parcelas vencidas no Supabase e enviar lembretes WhatsApp"
```

3. **`cargo-ai-digest`** — Resumo diário para gestores
```
   openclaw cron add \
     --name "Daily AI digest" \
     --cron "0 8 * * 1-5" \
     --tz "America/Sao_Paulo" \
     --message "Gerar resumo diário: cotações pendentes, OS em andamento, documentos financeiros, aprovações pendentes"
     --channel whatsapp
```

**Integração OpenClaw ↔ Supabase:**
- OpenClaw chama a Supabase REST API (PostgREST) para ler/escrever dados
- notification-hub enfileira mensagens WhatsApp na tabela `notification_logs`
- OpenClaw skill consome essas mensagens e envia via WhatsApp nativo
- Ou: workflow-orchestrator faz HTTP call direto para a OpenClaw API para envios imediatos
