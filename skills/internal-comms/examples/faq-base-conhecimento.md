# Diretriz: FAQ / Artigo da Base de Conhecimento

## Formato

Use este formato para documentar processos, responder a perguntas frequentes ou explicar um conceito técnico. O objetivo é criar uma fonte única de verdade (Single Source of Truth).

```markdown
# [Título Claro e Descritivo, ex: Como Funciona a Autenticação de Dois Fatores (2FA)]

**Público-alvo**: [ex: Desenvolvedores, Operações, Suporte]
**Última atualização**: [AAAA-MM-DD]

## Contexto

[Por que este documento existe? Qual problema ele resolve ou qual pergunta ele responde?]

## [Seção 1: Título Descritivo]

[Explicação detalhada. Use parágrafos, listas e blocos de código conforme necessário.]

### [Sub-seção 1.1]

[Detalhes adicionais.]

## [Seção 2: Passo a Passo (se aplicável)]

1.  **Passo 1**: [Descrição do passo]
2.  **Passo 2**: [Descrição do passo]

## Pontos de Atenção / Armadilhas Comuns

-   **Ponto de Atenção 1**: [Descrição]
-   **Ponto de Atenção 2**: [Descrição]

## Perguntas Frequentes Relacionadas

-   **P: [Pergunta]?**
    -   R: [Resposta concisa].
```

## Tom

-   **Didático e claro**: Assuma que o leitor tem algum conhecimento, mas não é um especialista no tópico.
-   **Estruturado**: Use títulos, subtítulos e listas para facilitar a leitura e a busca de informações.
-   **Prático**: Inclua exemplos de código, comandos ou screenshots sempre que possível.
