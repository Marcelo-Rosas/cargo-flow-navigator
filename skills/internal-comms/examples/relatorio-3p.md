# Diretriz: Relatório 3P (Progresso, Planos, Problemas)

## Formato

Use este formato para atualizações semanais de status de um projeto ou squad. O objetivo é ser conciso e direto.

```markdown
### Atualização 3P - [Nome do Projeto/Squad] - [Data]

**Progresso (Progress)**
- [Conquista 1]
- [Conquista 2]
- [Conquista 3]

**Planos (Plans)**
- [Próximo passo 1]
- [Próximo passo 2]
- [Próximo passo 3]

**Problemas (Problems)**
- [Obstáculo 1]
- [Obstáculo 2]
```

## Tom

- **Claro e conciso**: Use bullet points e frases curtas.
- **Orientado a fatos**: Foque no que foi feito, no que será feito e nos bloqueios. Evite linguagem vaga.
- **Transparente**: Seja honesto sobre os problemas. O objetivo é sinalizar a necessidade de ajuda.

## Coleta de Conteúdo

- **Progresso**: Revise os commits, PRs e tickets concluídos na última semana.
- **Planos**: Consulte o backlog ou o plano de projeto para as próximas tarefas.
- **Problemas**: Liste quaisquer dependências externas, débitos técnicos que estão atrasando o progresso ou decisões que precisam ser tomadas.
